# SBCBenchmark (консольный бенчмарк для Linux-одноплатников)

## Назначение

SBCBenchmark — консольное приложение на C для сценарного тестирования производительности и устойчивости одноплатных компьютеров (SBC) в Linux. Программа запускает набор нагрузок, собирает телеметрию, рассчитывает метрики и формирует отчёт по каждому запуску.

> Важно: длительные и интенсивные тесты могут заметно нагревать плату.

## Архитектура

Проект разделён на логические модули:

- `sbc_bench_scenarios.*` — каталог сценариев и меню;
- `sbc_bench_coordinator.*` — подготовка запуска и структура run-каталогов;
- `sbc_bench_workloads.*` — CPU / memory / jitter / NN-нагрузки;
- `sbc_bench_storage.*` — тест накопителя;
- `sbc_bench_network.*` — сетевой RTT-тест;
- `sbc_bench_telemetry.*` + сбор в `sbc_bench_v4.c` — телеметрия;
- `sbc_bench_metrics.*` — статистика и расчёт показателей;
- `sbc_bench_report.*` — сохранение артефактов и отчётов.

## Сценарии

Доступны сценарии:

- `baseline` — базовый тест;
- `long_soak` — длительный тест устойчивости;
- `server_gateway` — сценарий периферийного сервера;
- `iot` — сценарий контроллера интернета вещей;
- `embedded` — встраиваемый сценарий;
- `neural_host` — лёгкая имитация нейросетевой (матричной) нагрузки;
- `manual` — ручной выбор отдельных тестов.

## Описание тестов

- **CPU**: длительность, 1/многопоточный режим, подсчёт операций, производительность по временным окнам, деградация.
- **Memory**: read/write/copy и расчёт пропускной способности, безопасное ограничение буфера.
- **Storage**: seq write/read, random read, fsync, операции с файлами/каталогами, latency-метрики.
- **Network**: RTT через `ping` с детализацией по выборкам.
- **Jitter**: отклонение пробуждения таймера от ожидаемого, включая пороги >500 и >1000 мкс.
- **Telemetry**: температура, CPU freq (если доступно), CPU util, память, PSI.

Часть датчиков может быть недоступна на конкретной плате/ядре/дистрибутиве; это фиксируется как предупреждение и не должно аварийно завершать запуск.

## Метрики

Считаются (в зависимости от конкретного теста и объёма данных):

- среднее, медиана, p95, p99, p99.9, максимум;
- IOPS и throughput;
- коэффициент устойчивости во времени;
- процент деградации;
- признаки снижения частоты при росте температуры.

## Требования к системе

- Linux (рекомендуется ядро с `/proc/pressure/*` для PSI-метрик);
- доступ к `ping` в `PATH` для сетевого RTT-теста;
- доступ на чтение к `/proc/*` и `/sys/*` (temperature/cpufreq/hwmon, если доступны на плате);
- права на запись в каталог проекта для сохранения результатов в `runs_v4_c/`.

## Сборка

```bash
gcc -std=c11 -O2 -pthread \
  sbc_bench_v4.c sbc_bench_scenarios.c sbc_bench_workloads.c \
  sbc_bench_storage.c sbc_bench_network.c sbc_bench_metrics.c \
  sbc_bench_report.c sbc_bench_coordinator.c sbc_bench_noise.c \
  sbc_bench_telemetry.c -lm -o sbc_bench_v4
```
